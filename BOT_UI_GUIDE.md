# Инструкция По Боту И Панелям

Это подробная инструкция по тому, как пользоваться ботом прямо в Telegram: где находятся кнопки, как открыть `Owner Panel`, какие сценарии работают в личке и что доступно в группах.

## С чего начать

В личке с ботом:

1. Отправь `/start`
2. Откроется главное inline-меню
3. Если ты владелец, в главном меню будет кнопка `Owner Panel`

Если кнопки не видно:

- отправь `/start` ещё раз
- проверь, что пишешь именно от `OWNER_USER_ID`
- если бот только что обновлялся, нужен рестарт процесса

## Главное Меню

Есть два контура: публичный пользовательский и owner/admin.

Обычному пользователю доступны:

- `Мой профиль`
- `Топы`
- `Апелляции`

Владельцу дополнительно доступны:

- `Профиль`
- `Ачивки`
- `Топы`
- `Апелляции`
- `Справка`
- `Модерация апелляций`
- `Кабинет модерации`
- `Owner Panel`

## Owner Panel

`Owner Panel` это главная админ-панель по всем командам проекта.

Внутри есть разделы:

- `Runtime`
- `Git и логи`
- `Память и чаты`
- `Файлы и медиа`
- `Live-data`
- `Модерация`
- `Все команды`

## Раздел Runtime

Показывает текущее состояние runtime:

- owner report
- heartbeat
- backup state
- краткую сводку по ресурсам

Какие команды относятся сюда:

- `/status`
- `/ownerreport`
- `/resources`
- `/topproc`
- `/disk`
- `/net`
- `/restart`
- `/ownerautofix on|off|status`

`/restart` больше не перезапускает процесс изнутри runtime. Команда только сообщает, что self-restart отключён; если нужен реальный перезапуск, его делает внешний supervisor.

## Раздел Память И Чаты

Здесь собраны команды анализа истории и памяти:

- `/remember <факт>`
- `/recall [запрос]`
- `/search <запрос>`
- `/who_said <запрос>`
- `/history [@username|user_id]`
- `/daily [YYYY-MM-DD]`
- `/digest [YYYY-MM-DD]`
- `/chatdigest <chat_id> [YYYY-MM-DD]`
- `/export [chat|today|@username|user_id]`
- `/portrait [@username]`
- `/memorychat [запрос]`
- `/memoryuser [@username|user_id]`
- `/memorysummary`
- `/reset`

### Что важно

- `/digest` работает для текущего чата
- `/chatdigest` работает для конкретного `chat_id` из owner-лички
- `/history` и `/portrait` можно вызывать через reply на сообщение участника
- `/memorychat` показывает текущий chat memory слой
- `/memoryuser` показывает user memory по участнику текущего чата
- `/memorysummary` показывает summary memory snapshots

## Раздел Файлы И Медиа

Тут собраны файловые сценарии:

- `/sdls [/sdcard/путь]`
- `/sdsend /sdcard/путь/к/файлу`

Что умеет бот:

- анализировать фото
- анализировать документы
- вытаскивать excerpt из текстовых файлов
- использовать reply-контекст вокруг медиа

## Как Бот Работает В Группах

В группах bot не отвечает на всё подряд.

Обычно нужен один из триггеров:

- прямое обращение `Jarvis ...`
- прямое обращение `Enterprise ...`
- reply на сообщение бота
- упоминание бота
- некоторые owner-сценарии

Для owner в группе это теперь ещё и явный переключатель профиля:

- `Jarvis ...` — разговорный слой
- `Enterprise ...` — инженерный слой
- без имени сообщение owner в группе игнорируется

## Как Работает В Личке

В личке:

- обычный пользователь видит профиль, рейтинги и апелляции
- `Jarvis` отвечает владельцу как обычный режим
- `Enterprise` доступен владельцу как более исполнительный режим
- owner-команды и `Owner Panel` работают именно здесь

## Что Где Лучше Делать

В личке владельца:

- runtime
- git
- ошибки
- события
- digest по группам
- upgrade
- файловые операции

В группах:

- обычные ответы
- moderation по ситуации
- reply-aware ответы на конкретные сообщения
- live-data вопросы

## Если Что-То Не Видно

Если нет `Owner Panel`:

- отправь `/start`
- проверь, что ты владелец
- проверь, что bot перезапущен после обновления

Если нет реакции на кнопку:

- открой главное меню заново через `/start`
- проверь лог `tg_codex_bridge.log`
- проверь, что не упёрлись в старый runtime

Если нужна команда с параметрами:

- открывай нужный раздел панели
- там будет usage-подсказка
- сам вызов делается обычным текстом командой
