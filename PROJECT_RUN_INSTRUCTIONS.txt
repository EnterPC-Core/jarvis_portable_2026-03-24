Jarvis Project: Instructions

1. Main files
- tg_codex_bridge.py: main Telegram <-> Codex bridge
- run_jarvis_supervisor.sh: supervisor that keeps one bot process alive
- jarvis_memory.db: SQLite memory and service state
- local/whisper.cpp: local speech-to-text backend
- ../jarvis_legacy_data/jarvis.db: legacy database used only for rating, achievements and appeals

2. Start bot manually
cd /path/to/jarvis_portable_2026-03-24
cp .env.example .env
$EDITOR .env
BOT_TOKEN=your_token python tg_codex_bridge.py

3. Start via supervisor
cd /path/to/jarvis_portable_2026-03-24
sh run_jarvis_supervisor.sh

4. Stop bot
pkill -f 'python tg_codex_bridge.py'
pkill -f 'run_jarvis_supervisor.sh'

5. Syntax check
python -m py_compile tg_codex_bridge.py

6. Logs
- tg_codex_bridge.log
- tg_supervisor.out

7. Memory
- jarvis_memory.db
- jarvis_memory.db-wal
- jarvis_memory.db-shm
- legacy jarvis.db stays outside this project and is read by LEGACY_JARVIS_DB_PATH

8. Notes
- Bot expects one active instance only.
- Lock file: tg_codex_bridge.lock
- Voice recognition uses local whisper.cpp backend when available.
- Main runtime is tg_codex_bridge.py. Old legacy launchers should not be treated as the primary entrypoint.
- Legacy Jarvis DB is used only for achievements, rating, tops, chat stats and appeals through LegacyJarvisAdapter.
- Admin, moderation, warn and welcome functions live in tg_codex_bridge.py and jarvis_memory.db.
